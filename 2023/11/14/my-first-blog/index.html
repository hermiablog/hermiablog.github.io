HTML 面试题

一、语义化

  1. 说说你对 HTML 语义化的理解 ？HTML5 新增了哪些语义化标签 ？
  2. HTML5 新增的语义化标签

回答：

概念和示例

  • HTML 语义化指的是有英文语义的标签，比如<header本身就是头部的意思，可用于书写网页的头部，<table>本身就是表格的意思，所以用它来制作表格

    作用和原因

  • 语义化使开发者和机器更容易理解标签的含义
  • 在没有样式的时候，根据标签也能看懂页面的结构
  • 爬虫依赖于标签来确定上下文和关键词的权重，因此语义化有利于 SEO 优化，使爬虫能获得更多有效信息
  • 有利于帮助辅助技术更好的转译网页，例如无障碍阅读
  • 国际化，可以让各国开发者看懂
  • 提供可维护性，减少网页间差异
语义化知识点

概念：HTML语义化是指写html结构时用有英文语义的标签 作用：更易于开发人员和机器的阅读理解 示例：语义元素：<form>、<header>、<table> 非语义元素：div、span

为什么要语义化？

  • 为了在没有css样式的情况下，页面也能呈现出很好的内容结构、代码结构，易于阅读
  • 提高用户体验：例如alt属性用于图片内容说明
  • 有利于SEO优化：与搜索引擎建立良好沟通，有助于爬虫抓取更多的有效信息，爬虫依赖于标签来确定剩下文和各个关键字的权重
  • 可访问性：帮助辅助技术更好地转译网页，利于无障碍阅读（视障用户的屏幕阅读器）
  • 国际化：使用遵循W3C标准通用的语义化标签可以让各国开发者无障碍看懂我们网页的结构
  • 可维护性：语义化更具有可读性，减少网页之间的差异，方便后期维护

语义化实践中应注意什么

  • 尽量少使用无语义化的标签
  • 语义不明显时，既可以使用<div>或者<p>时，尽量使用<p>，因为p标签在默认情况下有上下间距，有利于兼容特殊终端
  • 不要使用纯样式标签，例如b/font/u等，要用css代替
  • 需要强调的文本，使用<strong>或<em>代替<b>和<i>
  • 写html、css、js……都要遵循用语义化标签，不随意取名，不要使用汉语拼音，不利于后期的代码重构和维护
  • 使用表格时，要区分表头<thead>、主体<tbody>、尾部<tfoot>,标题要用<caption>,表头用<tr><th>,单元格用<tr><td>
  • 表单域要用<fieldset>标签包裹，并用<legend>说明表单的用途 每个<input>标签对应的说明文本都要用<lable>标签，并且通过为<input>设置id属性，在<lable>标签中设置for = Id来让说明文本和相对应的input关联起来
